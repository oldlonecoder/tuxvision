U64 tux::ui::keyevent::code {0}

enough room for each BYTE ex [SHIFT+F5] bytes sequence: 0x1b,0x5b,0x31,0x35,0x3b,0x32,0x7e is compressed into 0x00005B31353B327E => 1b can be removed knowing it is a (ctrl)sequence. ...But we need to compact the bytes sequence into a u64 each time. so it needs to be verified if U64==U64 vs str comp is that faster so we can switch And maybe even enough to include data from incoming(as of sept 11,2024) support to mouse events.
